# QCUX 界面测试清单

这份清单覆盖最近一轮界面与交互改动。每条都包含执行步骤和预期结果，适合手工回归测试。

## 1. 启动与开发模式

### 1.1 网页开发模式
- 操作：执行 `npm run dev`
- 预期：只启动 Vite 开发服务器，不自动拉起 Electron 窗口；可直接在浏览器或 VS Code 内置浏览页访问应用

### 1.2 Electron 开发模式
- 操作：执行 `npm run electron:dev`
- 预期：启动渲染进程开发服务器并打开 Electron 桌面窗口

## 2. 保存与项目状态

### 2.1 未保存状态显示
- 操作：新建项目后修改项目名称、页面名称或任一元素属性
- 预期：窗口标题与工具栏右侧项目状态出现未保存圆点

### 2.2 全局快捷保存
- 操作：在交互设计界面按 `Ctrl+S` / `Cmd+S`
- 预期：触发保存逻辑；网页模式下载 `project.qcux.json`，Electron 模式按已有路径直接保存或首次弹出保存对话框

### 2.3 页面设计界面保存按钮
- 操作：进入界面设计界面后点击“💾 保存项目”
- 预期：与全局快捷保存逻辑一致，保存成功后未保存圆点消失

### 2.4 未保存保护
- 操作：制造未保存修改后尝试刷新网页或打开其他项目
- 预期：出现确认提示，防止直接丢失更改

## 3. 交互设计界面

### 3.1 页面选中与双击进入
- 操作：在交互设计界面单击某个页面，再双击该页面
- 预期：单击后页面边框高亮；双击后进入对应页面的界面设计界面

### 3.2 空白画布左键平移
- 操作：在交互设计界面的空白画布区域按住鼠标左键拖动
- 预期：视口平移；如果拖动起点不是页面或链接，不会错误触发页面移动或选择

### 3.3 工具栏窄宽度响应
- 操作：缩窄窗口宽度，观察交互设计顶部工具栏
- 预期：右侧项目状态仍保持可见；必要时出现“右侧还有按钮 →”提示；不会把 `Untitled` 或保存状态完全挤没

### 3.4 空状态 toolbox 项目名称编辑
- 操作：确保未选中页面和链接，在交互设计右侧 toolbox 修改项目名称
- 预期：项目名称立即更新，并标记为未保存

### 3.5 页面复制与粘贴
- 操作：选中一个页面后按 `Ctrl+C` / `Cmd+C`，再按 `Ctrl+V` / `Cmd+V`
- 预期：生成新页面；新页面在原页面右下偏移位置；新页面自动被选中；新页面和内部元素都使用新 ID；原页面的 links 不会被复制

### 3.6 链接替换确认
- 操作：为某个源元素先创建一个链接，再尝试把同一源元素链接到另一个目标页面
- 预期：弹出确认对话框；确认后删除旧链接并创建新链接；如果新目标与旧目标相同，则不重复创建链接

### 3.7 链接箭头方向与起点样式
- 操作：在不同方向上为页面创建链接
- 预期：起点显示为较小红点；箭头从目标页面最近侧边进入；箭头三角朝向正确

### 3.8 页面层级联动
- 操作：让两个页面缩略图互相覆盖，选中被遮挡的下层页面
- 预期：页面缩略图、标题、Landing 标记作为一个整体一起被遮挡或一起抬到最上层；不会出现标题漂浮在所有页面之上

## 4. 界面设计界面

### 4.1 未选中元素时编辑界面名称
- 操作：进入界面设计界面，点击空白取消元素选中，在右侧 toolbox 修改“界面名称”
- 预期：当前页面标题立即更新，并在返回交互设计界面后同步显示在页面缩略图标题中

### 4.2 添加元素后默认选中
- 操作：点击“➕ 添加物件”“🔤 添加文字”或“🔘 添加按钮”
- 预期：新元素被创建并立即处于选中状态，显示 8 个尺寸手柄

### 4.3 字号步进
- 操作：选中文字或按钮元素，连续点击 `A+` 和 `A-`
- 预期：字号每次增加或减少 4，而不是 1

### 4.4 删除元素快捷键
- 操作：选中任一元素后按 `Delete`
- 预期：元素被删除，toolbox 回到未选中状态

### 4.5 删除元素按钮
- 操作：选中任一元素后点击 toolbox 中“删除元素”
- 预期：元素被删除，当前选中状态被清空

### 4.6 元素复制与连续粘贴
- 操作：选中任一元素后按 `Ctrl+C` / `Cmd+C`，再连续按两次 `Ctrl+V` / `Cmd+V`
- 预期：第一次粘贴生成的新元素位于原元素右下偏移位置；第二次粘贴再继续向右下偏移；每次粘贴后自动选中新元素；新元素获得新的 ID 和更高 z-index

### 4.7 Text/Button 文字输入框宽度
- 操作：选中 Text 或 Button 元素，观察 toolbox 中“文字”输入框
- 预期：“文字”字段单独占一行，并撑满右侧 toolbox 可用宽度；不再是窄数字框样式

### 4.8 文字内容修改
- 操作：在 Text 或 Button 元素的“文字”输入框中输入较长文本
- 预期：文本可完整输入与编辑；元素内容同步更新；toolbox 布局不被挤坏

## 5. 导出与 Landing Page

### 5.1 Landing Page 标记
- 操作：创建多个页面，切换 Landing Page
- 预期：交互设计界面只有一个页面显示 Landing 标记，且标记无额外背景遮罩

### 5.2 导出网页
- 操作：点击“📤 导出”，用导出的 HTML 打开页面并点击存在交互的元素
- 预期：初始显示 Landing Page；点击有交互的元素会跳转到目标页面

## 6. 回归检查

### 6.1 TypeScript 检查
- 操作：执行 `npx tsc --noEmit`
- 预期：类型检查通过，无新增 TypeScript 错误

### 6.2 生产构建
- 操作：执行 `npm run build`
- 预期：前端与 Electron 构建成功完成
