
这篇文档描述 QCUX 当前版本的主要界面行为。

QCUX 是一个 UX 线框图设计工具，由两个主界面组成：交互设计界面和界面设计界面。默认进入交互设计界面，新项目未保存前显示为 `Untitled`。

交互设计界面包含画布、toolbar 和 toolbox。toolbar 中有打开项目、保存项目、导出、显示全局、新增界面、删除界面、设置 Landing Page 等按钮。点击新增界面会弹出对话框，对话框中包含界面标题、宽度、高度输入框，以及确定和取消按钮。点击确定会创建页面并跳转到界面设计界面。

交互设计界面中会显示页面缩略图、标题和 Landing 标记，这三部分按统一层级一起渲染与遮挡。选中页面时边框高亮，删除界面按钮和设置 Landing Page 按钮可用；点击空白区域会取消选中。用户可以拖动页面位置、双击页面进入界面设计，还可以用鼠标左键在空白画布上拖动来平移视口。显示全局会把全部页面按最大可用比例居中显示。

交互设计右侧 toolbox 在不同状态下显示不同内容。选中页面时显示页面信息和“添加交互”按钮；选中链接时显示链接信息和删除链接按钮；未选中任何对象时显示项目名称输入框，可直接修改项目名称。

界面设计界面包含画布、toolbar 和 toolbox。toolbar 中有添加物件、添加文字、添加按钮、保存项目、退出到交互设计界面等按钮。画布默认自适应当前页面大小，支持滚轮缩放和中键平移。新增元素后元素会立即被选中，并显示八个手柄用于改尺寸。

界面设计 toolbox 在未选中元素时显示界面名称输入框，可直接修改当前页面标题。选中 Object 时显示 X、Y、宽、高、名称和删除按钮；选中 Text 或 Button 时显示 X、Y、宽、高、全宽文字内容输入框、字体大小输入框、A-/A+ 按钮和删除按钮。字体按钮每次调整字号 4 点。

快捷操作方面，`Ctrl+S` / `Cmd+S` 可以在任意界面保存项目。交互设计界面中，选中页面后可用 `Ctrl+C` / `Ctrl+V` 复制页面，复制出的新页面会重新生成页面和元素 ID，不会复制原有链接，并出现在原页面右下偏移位置。界面设计界面中，选中元素后可用 `Delete` 删除元素，也可用 `Ctrl+C` / `Ctrl+V` 复制元素；连续粘贴时会持续向右下偏移，并自动选中新元素。

交互链接由红色折线箭头表示，起点是小红点，终点会根据目标页面相对方位自动选择最近的进入侧边。如果一个源元素已经有链接，再新增交互时会先询问是否替换旧链接；如果新旧目标相同，则保持原链接不变。

项目数据以 JSON 文件保存，包含 `pages` 列表与 `links` 列表。项目使用 TypeScript 开发，可在网页模式下直接调试，也可打包为 Electron 桌面程序。

Change
1. 交互设计支持空白画布左键拖拽平移视口。
2. 页面设计工具栏增加保存按钮，项目支持统一的 `Ctrl+S` / `Cmd+S` 保存。
3. 工具栏增加项目状态显示，未保存时显示圆点。
4. 交互设计空状态 toolbox 改为项目名称输入框。
5. 页面设计空状态 toolbox 改为界面名称输入框。
6. 页面设计支持元素删除、元素复制粘贴、连续偏移粘贴。
7. 交互设计支持页面复制粘贴，复制时不带链接。
8. 交互箭头更新为起点圆点、目标最近侧接入，并支持替换确认。
9. 页面缩略图、标题、Landing 标记统一层级渲染。
